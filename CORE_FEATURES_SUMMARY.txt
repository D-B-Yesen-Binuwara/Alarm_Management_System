CORE FEATURES AND IMPLEMENTATION LOGIC

1. Alarm Management System (Full CRUD)
What Covered: Full backend REST API stack (Controller, Service, Repository) to manage the network alarm lifecycle.
Implementation Logic: Uses EF Core against SQL Server to handle end-to-end CRUD operations, including device-specific filtering and automated timestamping.

2. Heartbeat Monitoring System
What Covered: A persistence layer and API for recording heartbeat histories, coupled with an automated generation scheduler.
Implementation Logic: An ASP.NET Core BackgroundService polls the database every 30 seconds, persisting a new "UP" heartbeat record for every active device to maintain a real-time pulse.

3. Failure Detection Engine
What Covered: An automated background service that acts as the core monitoring brain to identify timed-out devices.
Implementation Logic: The engine batch-loads the latest heartbeats and applies a strict 90-second timeout rule, transitioning devices to DOWN and logging timeout events if the threshold is breached.

4. Topology-Aware Failure Detection
What Covered: Intelligent network routing validation using the DeviceLink table to detect and prevent cascading alarm storms.
Implementation Logic: It evaluates multi-parent network paths; if all upstream parents of a device are DOWN or UNREACHABLE, the downstream device is marked UNREACHABLE instead of DOWN.

5. Simulation System (Controlled Failures)
What Covered: An API-driven deterministic testing mechanism using an `IsSimulatedDown` entity flag instead of random probabilities.
Implementation Logic: Triggering the simulation endpoint toggles the flag, instructing the HeartbeatSchedulerService to pause signal generation which natively triggers a failure timeout 90 seconds later.

6. Event Logging System
What Covered: An append-only audit repository dedicated exclusively to tracking critical simulation and monitoring state transitions.
Implementation Logic: Automatically persists "HEARTBEAT_TIMEOUT" or "HEARTBEAT_RECOVERED" events only when the failure engine detects genuine, non-topology status transitions.

7. Device Health API
What Covered: Diagnostic endpoints that supply frontend applications with real-time individual device health and network-wide uptime summaries.
Implementation Logic: The API computes device viability on-the-fly by checking if the latest heartbeat occurred within the 90-second window, returning a simple boolean flag.
